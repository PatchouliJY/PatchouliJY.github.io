Leetcode010：正则表达式匹配

发表于 2019-04-02 | 分类于 Leetcode题解
字数统计: 1.3k字 | 阅读时长 ≈ 6分钟

题目

给定一个字符串 (s) 和一个字符模式 (p)。实现支持 . 和 * 的正则表达式匹配。

匹配应该覆盖整个字符串 (s) ，而不是部分字符串。

说明：
·s 可能为空，且只包含从 a-z 的小写字母。
·p 可能为空，且只包含从 a-z 的小写字母，以及字符 . 和 *。

思路

题目上手后会有两种想法，动态规划以及回溯法，题目个人认为还是挺难的，思考了有两天，最后采用的动态规划完成。

设定dpResult[sLen+1][pLen+1]来判断正则匹配，若s能够匹配p，则此格值不为 0。

至于为什么要将dp的范围设置为[sLen+1][pLen+1]，因为需要进行判空处理，则在s开头设置一个空值，p开头设置一个空值，进行s = null，p = null时的匹配。

而后进行dp数组的初始化。

dpResult[0][0]代表字符串s和字符串p都是空串，空串和空串肯定是能匹配的，所以dpResult[0][0] = 1；当只有字符串p是空串时，这时的字符串s和字符串p肯定是无法匹配的，所以dpResult[0.....n][0] = 0；当只有字符串s是空串时，我们看下图的初始情况，“ab”，”acab”，其中“a”，将会变成一个空串，所以在dpResult[0][2] = 1 ，它是由dpResult[0][0]决定的，这是一个状态转移，即当`p[j-1] == ‘‘时，dpResult[0][j]=dpResult[0][j-2]`。

然后需要考虑状态转移。

1.当指针指向的两个字符相等时，即s[i-1]==p[j-1]或者p[j-1]=='.'（因为'.'可以匹配任意单字符），只有当dpResult[i-1][j-1]是匹配的，dpResult[i][j]才是匹配的。

状态转移方程为 dpResult[i][j] = dpResult[i-1][j-1]。

2.当指针指向字符串p的'*'时，即p[j-1]=='*'时，分两种情况

①s[i-1]!=p[j-2]&&p[j-2]!='.'，举个例子，也就是"kac","kb*ac"，说明此时"b*"应该是一个空串，当是空串的转移方程前面提到过，dpResult[i][j]=dpResult[i][j-2]。

②s[i-1]==p[j-2]||p[j-2]=='.'，此时的"*"可以匹配0个，一个，多个前面的元素。

a.当"*"匹配前面0个元素，也就是空串喽~dpResult[i][j]=dpResult[i][j-2]。

b.当"*"匹配前面一个元素时，也就是"a","a*"这种情况，只要字符串s”*”前面的匹配了就可以，状态转移为dpResult[i][j]=dpResult[i][j-1]。

c.当"*"匹配前面多个元素时，也就是"aaaa","a*"这种情况，状态转移为dpResult[i][j]=dpResult[i-1][j]。

状态转移方程为 dpResult[i][j] = dpResult[i - 1][j] + dpResult[i][j - 1] + dpResult[i][j - 2]。
